# MFE Boundary Review

Check:

- domain ownership;
- route ownership;
- public contract;
- shared dependencies;
- shell knowledge;
- cross-app imports;
- fallback;
- independent build/test.

Output:

- verdict;
- coupling risks;
- required contract changes;
- tests needed.
