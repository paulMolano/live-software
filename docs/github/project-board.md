# GitHub Project Board

## Name

Software Vida Roadmap

## Purpose

The Project is the operating board for planning, Codex execution and PR review.
It should organize issues by status, track, epic, readiness and guide/docs
impact.

## Required Fields

- Status
- Track
- Work type
- Size
- Codex ready
- Guide update

See [`project-fields.md`](project-fields.md) for field definitions,
classification rules and Codex execution rules.

## Required Views

- Backlog
- Sprint actual
- Arquitectura
- Frontend
- Backend
- Learning
- Review
- Done

## Campos

- Status
- Type
- Area
- Phase
- Priority
- Learning goal
- Estimate
- Owner
- Epics
- Needs refinement
- Guide required

## Track Views

- Platform
- Product
- Guide
- Agents
- Design
- Research

The concrete configuration for epics, sub-issues, filters and usage rules lives
in [`project-operating-model.md`](project-operating-model.md).
