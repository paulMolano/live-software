# GitHub Workflow

## Repositorio

Un único repo:

`software-vida`

## GitHub Project

Crear proyecto:

`Software Vida Roadmap`

## Campos

- Status: Todo / Ready / In Progress / Review / Done
- Type: feature / bug / chore / doc / spike / adr
- Area: shell / training-mfe / api / ui-kit / contracts / docs / infra
- Phase: 0 / 1 / 2 / 3 / future
- Priority: P0 / P1 / P2
- Learning goal: yes/no

## Issues

Cada issue debe tener:

- objetivo;
- contexto;
- criterios de aceptación;
- validación;
- restricciones;
- learning goal.

## Branches

Formato:

`feat/<area>-<short-name>`

## PRs

Pequeños y revisables.

Cada PR debe incluir:

- resumen;
- archivos cambiados;
- capturas si UI;
- comandos ejecutados;
- tests;
- riesgos.
